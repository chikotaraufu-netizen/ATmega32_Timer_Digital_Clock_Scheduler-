Project 8: Timer-Based Digital Clock and Task Scheduler System
ATmega32 System Function Covered:
Time delay and time measurement system.
Project Theme:
Timer programming, timekeeping, periodic task scheduling, embedded timing control,
and real-time system behavior.
Problem Statement
Many computer engineering systems need accurate timing without stopping the main
program. Examples include digital clocks, periodic sensor sampling, display refreshing,
communication timeouts, and scheduled control tasks. Software delay loops are
simple, but they waste processor time and become inaccurate when the program
grows. The ATmega32 timers allow the microcontroller to generate regular timing events
using hardware timer resources.
This project introduces students to timer-based timekeeping and periodic task
scheduling using the ATmega32.
Project Objective
To design and implement a timer-based digital clock and task scheduler system
using an ATmega32 timer interrupt. The system should maintain time in seconds,
minutes, and hours while also executing periodic tasks such as LED blinking and sensor
sampling indication.
Expected System Operation
The system should use a hardware timer to generate a regular time base, such as 1 ms,
10 ms, or 1 second. From this time base, the firmware should update a digital clock and
run scheduled tasks.

Extension Task
Students can extend the project by adding multiple software timers. For example, one
task runs every 500 ms, another every 2 seconds, and another every 10 seconds.
Students should implement this using timer-generated scheduler flags, not blocking
delay loops.
